Moxy (or moxy) is a programmable mock proxy. It is an HTTP proxy exposing web hooks that you can use in order to tell it what to do, and when to do it.

Setting Up

Moxy can be used as a system executable or a deployable web app.

Run gem install moxy and then you can run moxy from your terminal. A useful scenario for running as a system executable is having several instances for many scenarios and applications.

As a gem

$ moxy  # no arguments, default to localhost, 9292
$ moxy integration-notifier.dev.com 3000

Deploying as a webapp

$ git clone https://github.com/jondot/moxy
$ cd moxy; rackup
A good scenario for deploying as a web app is on a dedicated integration server. Moxy will run on any rack handler.



Using Moxy

The basic principle of using moxy goes about instructing moxy to stub your expected requests for you, and then setting it up as a HTTP proxy.

Telling Moxy How to Behave

You have many ways to program moxy. All of those must conclude to a POST to your http://moxy-host:port/__setup__ endpoint.

Here is a bonus: when you navigate to http://moxy-host:port/__setup__ in your browser, you end up in your moxy console. You can program and manage moxy using a sleek interface!.

Moxy's Language

Return an empty content with 200 HTTP status code. 
stub_request :get, "http://google.com"
Return boo hoo! as content.. 
stub_request(:get, "http://google.com").to_return(:body => "boo hoo!")
When POSTing to http://google.com return boo hoo! with 500 HTTP status code. 
stub_request(:get, "http://google.com").to_return(:body => "boo hoo!", :code => 500)
